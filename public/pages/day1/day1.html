What is Node

Node.js is an open-source, cross-platform, back-end JavaScript runtime environment that runs on the V8 engine and executes JavaScript code outside a web browser. Source: Wikipedia

Before in time JavaScript was only frontend code that could run in browsers. With V8 runtime JavaScript became independent of browsers. Now we can create scripts, and invoke them. Software and backends. Node is a runtime that wraps the scripts built on V8. Node in cli calls the REPL to run a file or a statement -read, evaluate, print, loop. All languages have REPLs, Python also has it, with highlighting and autocompletion.

Node is a platform that fills a particular need and understanding this is absolutely essential. You definitely don’t want to use Node.js for CPU-intensive operations; in fact, using it for heavy computation will annul nearly all of its advantages. Where Node really shines is in building fast, scalable network applications, as it’s capable of handling a huge number of simultaneous connections with high throughput, which equates to high scalability.
Source: Topal.com

JavaScript datatypes

JavaScript is a loosely typed -it doesn't convert the type until runtime- and a dynamic language -any variable can be assigned values of all types. The rimitive values are innmutable, String, Boolean, BigInt, number, null, undefined and objects which are collections of properties are mutable like arrays.

Mental Note: Always use hard equality check- ‘2 === 2’, it shows I understand the datatypes I'm using AND don't concatenate different datatypes.

JavaScript variables

Javascript variable identifiers: var, let, const or nothing. We should always declare JavaScript variables with var,let, or const. The var keyword is used in all JavaScript code from 1995 to 2015. The let and const keywords were added to JavaScript in 2015. Only if you want your code to run in an older browser, you must use var, as it has a global scope and so it’s not recommended. When to Use JavaScript const??? As a general rule you should always declare variables with const unless you think the value of the variable will change, then we should use let.

REST API conventions


GET /posts Retrieve all the ressources (blog posts) 200 OK / 404 NOT FOUND
GET /posts/{id} Retrieves a single resource (blog post) by id 200 OK / 404 NOT FOUND
POST /posts Create a new ressource (blog post) 201 CREATED / 400 BAD REQUEST
PUT /posts/{id} Update a ressource by id 204 NO CONTENT / 404 NOT FOUND
PATCH /posts/{id} Partially updates a ressource id 204 NO CONTENT / 404 NOT FOUND
DELETE /posts/{id} Deletes an existing ressource by id 204 NO CONTENT / 404 NOT FOUND
